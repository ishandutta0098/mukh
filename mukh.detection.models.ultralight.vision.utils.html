mukh.detection.models.ultralight.vision.utils.box_utils.convert_locations_to_boxes(locations, priors, center_variance, size_variance)[source]

Convert regressional location results of SSD into boxes in the form of (center_x, center_y, h, w).

The conversion:

$$predicted_center * center_variance =

rac {real_center - prior_center} {prior_hw}$$

$$exp(predicted_hw * size_variance) =

rac {real_hw} {prior_hw}$$

We do it in the inverse direction here. Args:

locations (batch_size, num_priors, 4): the regression output of SSD. It will contain the outputs as well. priors (num_priors, 4) or (batch_size/1, num_priors, 4): prior boxes. center_variance: a float used to change the scale of center. size_variance: a float used to change of scale of size.

Returns:
boxes: priors: [[center_x, center_y, h, w]]. All the values

are relative to the image size.

mukh.detection.models.ultralight.vision.utils.box_utils.hard_negative_mining(loss, labels, neg_pos_ratio)[source]

It used to suppress the presence of a large number of negative prediction. It works on image level not batch level. For any example/image, it keeps all the positive predictions and

cut the number of negative predictions to make sure the ratio between the negative examples and positive examples is no more the given ratio for an image.

Parameters:

  • loss (N, num_priors) – the loss for each example.

  • labels (N, num_priors) – the labels.

  • neg_pos_ratio – the ratio between the negative examples and positive examples.
